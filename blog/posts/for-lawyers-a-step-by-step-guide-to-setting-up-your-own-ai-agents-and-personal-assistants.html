For Lawyers: A Step-by-Step Guide to Setting Up Your Own AI Agents and Personal Assistants

2026-06-29 · Legal Tech AI Productivity Guide

Introduction

Imagine having a team of assistants working around the clock. Researching case law, drafting documents, managing your calendar, tracking deadlines, creating content, and handling client communications. All of it. Without hiring a single extra person.

That is not a fantasy. That is PicoClaw.

PicoClaw is an ultra-lightweight, self-hosted AI assistant framework written in Go. It is designed to be practical, private, and extensible. You run it on your own machine (or a cheap VPS), connect it to Telegram, and suddenly you have a personal AI assistant that understands your practice, your cases, and your workflow.

This guide walks through everything. Installation, configuration, the tools and skills that matter most for a legal practice, and the real lessons learned from running it day-to-day.

---

Why a Lawyer Needs This

Let us be honest about what a typical day looks like:

  • You are juggling multiple cases with different courts, deadlines, and opposing counsel
  • Research means digging through statutes, case law, and regulations
  • Drafting documents requires precise formatting and proper citations
  • Client communication happens across WhatsApp, email, phone calls
  • You need to bill for your time but tracking it is a chore
  • Content marketing? Who has the time?

PicoClaw does not replace your judgment. It replaces the drudgery. The repetitive tasks. The items that pile up.

---

What You Will Need

Before we start, here is the shopping list:

  • A Linux machine. Any old laptop, a Raspberry Pi, or a cheap VPS will do.
  • Telegram account. Free, and the bot API is excellent.
  • Go 1.21+. The runtime.
  • Python 3. For skill scripts.
  • Git. For version control.
  • About an hour. To get everything running.

---

Step 1: Install PicoClaw

PicoClaw is distributed as a single binary. Installation is straightforward:

# Download the latest release for your platform
wget https://github.com/your-org/picoclaw/releases/latest/download/picoclaw-linux-amd64.tar.gz

# Extract
tar xzf picoclaw-linux-amd64.tar.gz

# Move to your PATH
sudo mv picoclaw /usr/local/bin/

Alternatively, if you have Go installed, you can build from source:

git clone https://github.com/your-org/picoclaw.git
cd picoclaw
go build -o picoclaw .
sudo mv picoclaw /usr/local/bin/

Verify the installation:

picoclaw version

You should see the version number and build info.

---

Step 2: Create a Telegram Bot

This is the easiest part. Telegram bots are free and instant to set up.

  1. Open Telegram and search for **@BotFather**
  2. Send /newbot
  3. Choose a name (e.g., "My Legal Assistant")
  4. Choose a username (must end in bot, e.g., my_legal_assistant_bot)
  5. BotFather will give you a **token** — save this immediately

    6. The token looks like: 123456789:ABCdefGHIjklmNOPqrstUVwxyz

    This token is how PicoClaw connects to Telegram. Keep it private.

    ---

    Step 3: Configure PicoClaw

    Create a configuration file. PicoClaw uses YAML for configuration.

    # config.yaml
name: "PicoClaw"

# Telegram configuration
telegram:
  token: "123456789:ABCdefGHIjklmNOPqrstUVwxyz"
  chat_id: "your-chat-id"  # Your personal chat ID

# Workspace
workspace: "/home/you/picoclaw-workspace"

# Skills directory
skills_dir: "/home/you/picoclaw-workspace/skills"

# Memory
memory_file: "/home/you/picoclaw-workspace/memory/MEMORY.md"

    To find your Telegram chat ID, send a message to your bot, then visit:

    https://api.telegram.org/bot/getUpdates

    Look for "chat":{"id": in the response. That is your chat ID.

    ---

    Step 4: Set Up the Workspace

    PicoClaw organizes everything in a workspace directory:

    mkdir -p ~/picoclaw-workspace
cd ~/picoclaw-workspace

# Create the directory structure
mkdir -p memory
mkdir -p skills
mkdir -p shared

    The workspace structure looks like this:

    picoclaw-workspace/
  config.yaml              # Main configuration
  memory/
    MEMORY.md            # Long-term memory
    YYYYMM/
      YYYYMMDD.md      # Daily notes
  skills/                  # Installed skills
    maigai-legal-research/
    sewa/
    ushe-personal-assistant/
    site-publisher/
    weather/
  shared/                  # Shared resources
    mailman.py           # Email utility
    reports/
  vitowebsite/             # Optional: static site

    ---

    Step 5: Run PicoClaw

    picoclaw --config config.yaml

    That is it. Your bot is now live. Send a message to your Telegram bot and it should respond.

    For production use, you will want to run it as a service:

    # Create a systemd service
sudo nano /etc/systemd/system/picoclaw.service

    [Unit]
Description=PicoClaw AI Assistant
After=network.target

[Service]
ExecStart=/usr/local/bin/picoclaw --config /home/you/picoclaw-workspace/config.yaml
WorkingDirectory=/home/you/picoclaw-workspace
Restart=always
User=you

[Install]
WantedBy=multi-user.target

    sudo systemctl enable picoclaw
sudo systemctl start picoclaw
sudo systemctl status picoclaw

    Step 6: Install Essential Skills

    Skills are what make PicoClaw useful. They are modular extensions that add capabilities. Here are the ones that matter most for a legal practice.

    6.1 Legal Research Skill (MaiGai)

    This is the workhorse. It searches your legal library using keyword-based RAG (Retrieval-Augmented Generation) to find relevant statutes, case law, and commentary.

    What it does:

    • Searches a library of legal documents (PDFs, markdown files, DOCs)
    • Returns relevant excerpts with citations
    • Can research topics across maritime law, trade law, petroleum law, and general Nigerian law
    • Tracks research tasks and reports

    Commands:

    • `research ` — Conduct legal research on a topic
    • `search ` — Quick keyword search of the library
    • `tasks` — Check pending research tasks

    How to set it up:

    # Install the skill
cp -r maigai-legal-research ~/picoclaw-workspace/skills/

# Build your legal library
mkdir -p ~/picoclaw-workspace/shared/law-library/
# Add your PDFs, statutes, case law documents here

# Index the library
python3 ~/picoclaw-workspace/skills/maigai-legal-research/scripts/index.py

    The RAG Index System:

    The legal library uses a pre-built index for fast searching. The system:

    1. Scans all documents in your law library
    2. Chunks them into searchable segments
    3. Builds a keyword index
    4. Auto-updates when files change

      5. For auto-indexing, set up a cron job:

      crontab -e
# Add:
0 * * * * cd ~/picoclaw-workspace && bash auto-index.sh

      This reindexes every hour. You can also run it manually:

      bash auto-index.sh        # Check and reindex if changed
bash auto-index.sh force  # Force rebuild all
bash auto-index.sh status # Show index health
